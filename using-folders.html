using folders

folders

Folders are a nice feature of gollum, which unfortunately cannot be accessed through the web interface.

folders and wiki links

First thing to note is that wiki pages linking works as if there were no folders. This may seem strange, but in practice is a nice way of not having broken links while being able to reorganize things freely. Links work as follows:

  • the only thing that you can include between square brackets [[...]] is a page name. Anything you put there will be interpreted as such. No way to refer to the folder structure. If you add a link such as [[buh/blah]] this will look for a page named buh--blah or something like that.
  • link resolution walks breadth-first the folder structure from the top, and picks the first page which matches.

why folders, then?

you'd be asking; well, the answer is sidebars and footers. You can put a sidebar at it will show up in any page located under the folder, and in those nested folders for which there's no sidebar (same for footers). That's a nice, practical way of creating TOCs and such.
